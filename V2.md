# V2 — What's New

V2 is a structural and content overhaul of the UX skill set. The goals: make
the skills actually discoverable by Claude Code, make them procedural instead
of purely reference material, and bring the guidance up to date (WCAG 2.2,
platform clipboard privacy, Core Web Vitals).

**16 skills → 18 skills.**

---

## Breaking Changes

### Folder renames (required for skill discovery)

Every skill folder is now prefixed `ux-` so the folder name matches the
`name:` field in its frontmatter — Claude Code treats the directory name as
the skill's identity, so a mismatch breaks discovery.

| `search` | `ux-search` |

### Fixed installation instructions

The V1 README instructed copying the whole repo to `.claude/skills/ux/`,
which nested every skill two levels deep (`.claude/skills/ux/general/SKILL.md`).
Claude Code only discovers skills one level deep, so **none of the V1 skills
were ever auto-discovered when installed as documented**. It also copied
`.git`, `LICENSE`, and `README.md` into the skills directory.

V2 install copies each skill folder directly into `.claude/skills/`:

```bash
cp -r design-skills/ux-* your-project/.claude/skills/
```

If you installed V1, delete `.claude/skills/ux/` and reinstall.

## New Skills

### `ux-review` — UX Review router

The missing entry point. A real review ("review this onboarding screen") spans
visual design, accessibility, content, and error handling at once, but nothing
in V1 told Claude to load more than one skill. `ux-review` maps the request
type to the category skills to read (a routing table) and defines a shared
output format: summary, per-category Pass/Fail/N/A findings with evidence and
severity, and top ranked fixes.

### `ux-notifications` — Notifications

In V1, notification guidance was scattered across four skills (general,
settings, safety-privacy, user-account) with no dedicated home. The new skill
covers permission timing, pre-permission prompts, per-type granularity,
transactional vs promotional separation, quiet hours, badge accuracy,
deep-link targets, Android channels, iOS provisional authorization, frequency
capping, and notification copy.

## Changes to Every Skill

- **"How to Review" section** — each skill now instructs Claude to mark every
  checklist item Pass / Fail / N/A, cite specific evidence, tag failures with
  a severity (Blocker / Major / Minor), and end with the top 3 fixes. V1 was
  reference material; V2 defines the review behavior.
- **"Related Skills" section** — cross-references so multi-category concerns
  (forms ↔ error handling ↔ content, etc.) load the right siblings.
- **Sharper descriptions** — every frontmatter description now leads with the
  distinctive keywords of its category instead of the identical
  "UX checklist and best practices for…" opener, improving skill selection
  when several descriptions compete for relevance.

## Per-Skill Content Changes

### `ux-general`
- Added a **Priority column** (Required / Recommended / Optional) to the
  checklist. The V1 claim that failing *any* item made the app "not ready for
  production" was wrong — widget support and SD-card storage are not launch
  blockers. Only Required items are now framed as table stakes.
- **Received** five items that were miscategorized under user-account:
  in-app operation status, cancel any action, pull-to-refresh, ad removal
  option, purchasable features.
- **Moved out** "In-app language" and "Clear cache" to `ux-settings`, where
  they belong.

### `ux-user-account`
- Removed the **duplicate "Password manager support"** item (it appeared
  twice) and merged both rows into one.
- Moved six non-account items out: in-app operation status, cancel any action,
  pull-to-refresh, ad removal, purchasable features (→ `ux-general`) and
  10-second comprehension (→ `ux-help-onboarding`).

### `ux-accessibility`
- Updated from **WCAG 2.1 to WCAG 2.2** (the standard since October 2023) and
  noted the **European Accessibility Act** (in force since June 2025).
- Fixed the touch-target citation: the AA criterion is **2.5.8 Target Size
  (Minimum), 24×24 px** — the previously cited 2.5.5 (44px) is AAA. The
  44pt / 48dp numbers come from Apple HIG and Material, not WCAG, and are now
  labeled as such.
- Noted that 2.3.3 (reduce motion) is **AAA**, not a compliance requirement.
- Added a **"New in WCAG 2.2"** table (Focus Not Obscured, Dragging Movements,
  Target Size, Consistent Help, Redundant Entry, Accessible Authentication)
  and new checklist rows for Focus Not Obscured, Accessible Authentication,
  and Redundant Entry.

### `ux-visual-design`
- Fixed the internal contradiction between "body text minimum 16sp" and the
  scale's 15sp/13sp/12sp roles: the minimum now explicitly applies to body
  copy, with smaller roles labeled supporting text.
- The typography scale is now labeled **iOS (HIG-derived)** and a **Material 3
  scale** was added, with guidance to use each platform's native scale.
- Removed "Minimum touch target" as the description of the 48pt spacing
  token — hit areas are not spacing tokens; a clarifying note was added.

### `ux-utility`
- **In-app browser guidance softened**: V1 said "never the system browser."
  V2 requires a visible "Open in browser" action and respecting the user's
  default browser — never trapping users in the in-app view.
- **Clipboard detection updated for platform privacy changes**: iOS 16+
  shows a system paste prompt on clipboard reads — use
  `UIPasteboard.detectPatterns` to detect URLs without reading contents.
  Android 12+ shows a system toast on clipboard access.

### `ux-network`
- Replaced the stale performance table (First Contentful Paint / Time to
  Interactive — TTI is deprecated) with **Core Web Vitals** for web
  (LCP ≤ 2.5s, INP ≤ 200ms, CLS ≤ 0.1) and a separate **mobile app** table
  (cold start, P95 API latency, cache hit, frame drops).

### `ux-error-handling`
- Now the **canonical home for empty-state standards**: a new section defines
  the four-part anatomy (visual, heading, message, action) and four variants
  (first use, no results, error-caused, cleared/completed). `ux-search`,
  `ux-visual-design`, `ux-content`, and `ux-help-onboarding` now reference it
  instead of each defining empty states slightly differently.

### `ux-settings`
- Added **"In-app language"** and **"Clear cache"** (moved from `ux-general`)
  plus a Storage section in the standard settings tree.
- Notification granularity row now points to `ux-notifications`.

### `ux-help-onboarding`
- Added **"10-second comprehension"** (moved from user-account, where it
  never belonged).
- Blank-slate pattern references the canonical empty-state standards.

### `ux-search`, `ux-forms`, `ux-navigation`, `ux-information-architecture`, `ux-content`, `ux-safety-privacy`, `ux-ai-automation`
- Description rewrites, How to Review + Related Skills sections, and
  cross-reference updates (e.g. forms now cites WCAG 2.2's Redundant Entry
  rule; content and search point at the canonical empty-state standards).
